# 테스트

### 정의

- 테스트는, 작성한 코드가 정상적으로 동작한다 라는 것을 증명하는 방법.

### 왜?

- 자동화된 테스트

### 어떻게?

- Main 메소드에서 프로그램을 실행해서 확인한다.
- 웹을 실행해서, 화면이 제대로 보이는지 눈으로 본다.
- 데이터베이스이면, 그 행이 있는지 확인한다.

- 위 사례의 단점
    - 눈으로 확인해야한다.

### 중요한점?

- 자동화
- 코드의 영향도를 파악하기 위해서
  - 내 코드가 정상동작한다는걸 증명
  - 코드가 변경됐을 때 영향 범위를 확인할 수 있다.
  - 그에 적절한 대응을 그때그때 할 수 있다.

- 무의미한 테스트는 오히려 독이될 수 있다.
